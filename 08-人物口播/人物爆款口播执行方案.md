# 人物爆款口播执行方案

这不是模板库。

这是以后做视频号人物口播时，每一条都要执行的生产流程。

目标：

> 不是模仿爆款文案，而是把爆款样本背后的方法，变成我们每天可以复制执行的动作。

## 一、我们从爆款样本里学到的核心

爆款人物视频不是靠资料多。

也不是靠把一个人讲完整。

真正能留人的人物口播，通常只抓一个核心：

```text
这个人身上最反常、最刺痛、最有命运感的那个瞬间。
```

比如：

- 胡适样本：不是讲胡适一生，而是抓“父亲写给儿子的反常识家书”。
- 沈安娜样本：不是讲所有谍战经历，而是抓“潜伏14年，从未暴露”。
- 秋瑾样本：不是讲革命履历，而是抓“本来富贵安稳，却主动走向死亡”。
- 让娜·卡尔芒样本：不是讲长寿知识，而是抓“反常识生活方式 + 活成时间见证者”。

所以我们以后做人物口播，第一步永远不是写稿。

第一步是找：

> 这个人物最值得被点开的那一刻。

## 二、选题执行标准

以后拿到一个人物，先打分。

满分100分。

低于75分，不写。

### 1. 命运反差：30分

看这个人有没有强烈反差。

高分人物通常具备：

- 本来可以安稳，却选择危险
- 本来身份很高，却主动放弃
- 本来没人看好，后来改变命运
- 表面普通，背后藏着巨大秘密
- 活法和普通认知完全相反

### 2. 关键选择：20分

看这个人有没有一个能讲清楚的选择。

例如：

- 她选择不嫁
- 她选择革命
- 他选择写下那封信
- 她选择签下那份房产协议
- 他选择不低头

没有选择，就没有故事。

### 3. 代价：20分

看这个选择有没有代价。

代价可以是：

- 死亡
- 孤独
- 失去身份
- 失去家庭
- 被误解
- 多年隐忍
- 一生背负秘密

没有代价，人物不重。

### 4. 画面感：15分

看能不能拍出画面。

比如：

- 一封信
- 一张旧照片
- 一间空房子
- 一个临终画面
- 一场审判
- 一次潜伏现场
- 一张长桌旁只剩一个老人

没有画面，视频不好做。

### 5. 评论潜力：15分

看观众听完会不会想说话。

常见评论点：

- 如果是你会怎么选？
- 她这样值不值？
- 这算清醒还是固执？
- 长寿到底是福还是苦？
- 父母和孩子到底该不该互相亏欠？

没有评论点，传播会弱。

## 三、人物口播生产流程

每一条人物口播都按下面流程走。

### 第一步：一句话定位人物

不要写履历。

先用一句话说清楚这个人为什么值得讲。

格式：

```text
他/她是一个{身份}，但真正让人记住的，不是{表面成就}，而是{命运反差/关键选择/代价}。
```

示例：

```text
秋瑾是一个本来可以享尽富贵的女人，但真正让人记住的，是她偏偏抛下这一切，走向一条会死的路。
```

### 第二步：找到唯一主钩子

一条视频只抓一个主钩子。

不要贪多。

主钩子必须能放在开头3秒。

常见主钩子类型：

1. 富贵反差
2. 潜伏秘密
3. 临终一句话
4. 家书反常识
5. 年龄数字冲击
6. 命运荒诞交易
7. 被误解多年
8. 高光背后代价

### 第三步：确定情绪路线

人物口播不是一直燃，也不是一直惨。

要有情绪路线。

常用路线：

```text
惊讶 → 好奇 → 紧张 → 心疼 → 沉默
```

```text
震惊 → 反差 → 代价 → 敬佩 → 余味
```

```text
好笑 → 荒诞 → 反转 → 孤独 → 沉默
```

写稿前必须先选一条路线。

### 第四步：写开头

开头只能做一件事：让观众停下。

开头不要介绍人物。

开头要直接抛命运。

合格开头必须具备至少两个元素：

- 强数字
- 强反差
- 强身份
- 强结局
- 强选择
- 强代价

不合格开头：

```text
今天我们来讲一位伟大的人物。
```

合格开头：

```text
她住着148间房的豪宅，丈夫在朝廷当官，自己还是诰命夫人。可她偏偏抛下一切去革命，32岁就走到了刑场。
```

### 第五步：正文只保留5个段落

90秒人物口播，不要写太多。

只保留5段。

```text
第1段：命运反差
第2段：人物原本可以怎样活
第3段：人物做了什么选择
第4段：这个选择带来什么代价
第5段：最后一个画面或一句话收住
```

如果超过5段，容易散。

### 第六步：每段只完成一个任务

不要一段里同时讲背景、成就、评价、结局。

每段只干一件事。

例如：

- 第一段只负责抓人
- 第二段只负责抬高原本人生
- 第三段只负责转折选择
- 第四段只负责代价加重
- 第五段只负责情绪落点

### 第七步：删掉解释腔

人物口播不能跳出来分析。

禁止出现：

- 这说明了什么
- 我们应该看到
- 这才是她伟大的地方
- 所以讲这个人物不能只讲她勇敢
- 这个故事告诉我们

替代方法：

用动作、场面、后果替代评价。

不要说：

```text
她很勇敢。
```

要写：

```text
她知道这一去可能回不来，还是把家门关上，走了出去。
```

## 四、标准成稿结构

以后我们生成最终稿，一律使用这个结构。

```text
标题：

开头钩子：

正文：

【第一段】命运反差
【第二段】原本人生
【第三段】关键选择
【第四段】代价加重
【第五段】余味收尾

画面执行：
1.
2.
3.
4.
5.

字幕重点句：
1.
2.
3.

评论引导：

发布前检查：
```

注意：

最终稿不是给作者看的文章，而是给口播者直接念的稿。

必须短句。

必须能念。

必须有停顿。

## 五、画面执行方案

人物口播不能只写文案。

每条都要同步给画面方案。

### 画面原则

1. 竖屏9:16
2. 前3秒必须出现强字幕
3. 人物脸部近景优先
4. 旧时代人物用复古、黑白、泛黄质感
5. 情绪越重，画面越简单
6. 不要堆太多图
7. 一个画面只服务一句重点话

### 5段画面对应法

```text
第1段：强反差画面
第2段：人物原本生活画面
第3段：关键选择画面
第4段：代价/危险/孤独画面
第5段：人物背影/旧照/空镜收尾
```

## 六、标题执行方案

标题先写20个，再选3个，最后定1个。

标题不是越文艺越好。

标题要先让人点。

### 高点击标题必须包含一个爆点

- 年龄
- 身份
- 死亡
- 秘密
- 反差
- 代价
- 一句话
- 一封信
- 潜伏年数
- 荒诞交易

### 标题筛选标准

每个标题按5项打分：

1. 点击欲望
2. 人物反差
3. 情绪冲击
4. 信息清晰
5. 视频号适配

低于4星不用。

## 七、发布前执行清单

每条发布前必须问：

1. 前3秒能不能让人停下？
2. 这条视频只讲一个主钩子吗？
3. 有没有人物命运反差？
4. 有没有关键选择？
5. 有没有代价？
6. 有没有具体画面？
7. 有没有作者跳出来评价？
8. 能不能直接念？
9. 结尾有没有余味？
10. 评论区有没有话题？

10项里少于8项通过，不发布。

## 八、以后对 ChatGPT 的执行口令

以后不用说“套模板”。

直接这样下指令：

```text
按人物爆款口播执行方案，分析这个人物是否值得写。先打分，再找主钩子，再给20个标题，最后推荐1个最适合视频号的角度。
```

选好后说：

```text
按人物爆款口播执行方案，直接生成最终发布版。要求包括标题、口播正文、画面执行、字幕重点句、评论引导、发布前检查。
```

如果你给的是视频样本，就说：

```text
参考这个视频样本，不要套用原文，提炼它的爆款机制，然后转成我们可执行的人物口播生产动作。
```

## 九、我们自己的执行原则

参考爆款，不是复制爆款。

以后每分析一个爆款视频，都必须输出三个结果：

1. 它为什么能留人
2. 它背后的可复用机制是什么
3. 我们下一条视频具体怎么执行

只拆文案不够。

只给提示词也不够。

最终必须落到动作：

```text
选什么人物
抓哪个钩子
用什么标题
写哪5段
配哪5个画面
用哪句结尾
引导什么评论
```

这才是这个仓库真正要沉淀的东西。
